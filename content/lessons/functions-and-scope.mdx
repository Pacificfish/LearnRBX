# Functions & Scope

Functions are reusable blocks of code that perform specific tasks. They're essential for organizing your Roblox scripts!

## Creating Functions

Basic function syntax:

```lua
local function greet()
  print("Hello!")
end

greet()  -- Call the function
-- Output: Hello!
```

## Functions with Parameters

Pass data into functions using parameters:

```lua
local function greet(name)
  print("Hello, " .. name .. "!")
end

greet("Alex")   -- Output: Hello, Alex!
greet("Jordan") -- Output: Hello, Jordan!
```

Multiple parameters:

```lua
local function addNumbers(a, b)
  local sum = a + b
  print("Sum: " .. tostring(sum))
end

addNumbers(5, 3)  -- Output: Sum: 8
```

## Return Values

Functions can return values:

```lua
local function multiply(x, y)
  return x * y
end

local result = multiply(4, 5)
print(result)  -- Output: 20
```

Multiple return values:

```lua
local function getPlayerInfo()
  return "Alex", 100, 5
end

local name, health, level = getPlayerInfo()
print(name .. " has " .. tostring(health) .. " health")
```

## Variable Scope

Understanding scope is crucial for writing bug-free code!

### Local Scope

Variables created with `local` only exist in their block:

```lua
local function test()
  local x = 10  -- Only exists inside this function
  print(x)
end

test()   -- Output: 10
-- print(x)  -- ERROR! x doesn't exist here
```

### Global Scope

Variables without `local` are global (avoid these!):

```lua
function badExample()
  globalVar = "I'm global"  -- DON'T DO THIS!
end
```

**Why avoid globals?**
- Can cause naming conflicts
- Harder to debug
- Worse performance
- Bad practice in Roblox

## Nested Functions and Closures

Functions can access variables from their parent scope:

```lua
local function createCounter()
  local count = 0  -- Upvalue
  
  return function()
    count = count + 1
    return count
  end
end

local counter = createCounter()
print(counter())  -- Output: 1
print(counter())  -- Output: 2
print(counter())  -- Output: 3
```

This is called a **closure** - the inner function "closes over" the `count` variable.

## Roblox Examples

### Damage Function

```lua
local function dealDamage(player, amount)
  local character = player.Character
  if character then
    local humanoid = character:FindFirstChild("Humanoid")
    if humanoid then
      humanoid.Health = humanoid.Health - amount
    end
  end
end
```

### Button Click Handler

```lua
local button = script.Parent

local function onButtonClick()
  print("Button was clicked!")
  -- Do something cool
end

button.MouseButton1Click:Connect(onButtonClick)
```

## Best Practices

1. **Always use `local`** for functions and variables
2. **Keep functions small** - each function should do one thing well
3. **Use descriptive names** - `calculateDamage()` is better than `calc()`
4. **Document complex functions** with comments
